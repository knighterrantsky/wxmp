# WeChat Private Media Upload V1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a locally runnable WeChat mini program and Fastify backend that authenticates WeChat users, records the confirmed nickname mapping, uploads private image/video files to Cloudflare R2 in fixed multipart chunks, shows real progress, and lists upload records through the terminal `uploaded` state.

**Architecture:** A pnpm monorepo contains a native TypeScript mini program, a Fastify TypeScript API, and a shared contracts package. The mini program only calls the HTTPS API; the API owns PostgreSQL and private R2 credentials and exposes dependency-injected WeChat and object-storage adapters so the complete flow is testable without real external credentials. PostgreSQL is the durable source of identity, multipart, finalizer, history, idempotency, and audit state.

**Tech Stack:** Node.js 24 LTS, pnpm workspaces, TypeScript strict mode, Fastify 5, TypeBox JSON Schema, PostgreSQL 17 with `pg`, AWS SDK v3 S3 client for R2 and local MinIO, `jose` Ed25519 JWT, `uuid` v7 IDs, `@noble/hashes` SHA-256, native WeChat mini program APIs, Vitest, Docker Compose, ESLint, and Prettier.

## Global Constraints

- Use the approved architecture in `docs/superpowers/specs/2026-07-14-wechat-private-media-upload-design.md`, API contract in `docs/api/media-upload-api.md`, and schema in `docs/database/media-upload-database.md`.
- A single file is `12..209715200` bytes; a normal part is exactly `8388608` bytes; a file has `1..25` parts; one selection contains at most 9 image/video files.
- Public upload status is exactly `uploading | finalizing | cancelling | uploaded | upload_failed | aborted | expired`; `uploaded` is the success terminal state.
- The R2 bucket is private. The mini program never receives R2 credentials, bucket, object key, multipart ID, ETag, signed URL, or a content-read endpoint.
- The R2 object key is generated by the backend as `users/{userId}/{image|video}/{yyyy}/{mm}/{mediaId}{canonicalExtension}` and never contains openid, nickname, or the original path.
- WeChat identity uniqueness is `(provider, app_id, openid)` and the client never receives `openid`, `unionid`, or `session_key`.
- Application logs and audit metadata never contain nickname text, openid, object keys, tokens, R2 credentials, internal ETags, or file bytes.
- Nickname capture uses user-confirmed `<input type="nickname">`; it is mutable display data and never an authorization input.
- `POST /v1/uploads` is called only after the user confirms the selected files a second time.
- R2 completion is asynchronous and durable: `complete` returns `202 finalizing`; a persistent finalizer converges via HEAD/ListParts/Complete/HEAD and projects success as `uploaded`.
- QNAP/NAS synchronization, collector jobs, public downloads, preview, share, and user deletion are outside this plan.
- Local and CI multipart integration uses a private MinIO bucket through the same S3 adapter; production configuration rejects the development WeChat stub, MinIO endpoints, and temporary signing keys.
- Production code follows red-green-refactor. Every new domain function, service, route, and adapter behavior must have a test that was observed failing for the intended reason before implementation.

```ts
export const MIN_FILE_SIZE_BYTES = 12
export const MAX_FILE_SIZE_BYTES = 209_715_200
export const PART_SIZE_BYTES = 8_388_608
export const MAX_SELECTION_COUNT = 9

export const PUBLIC_UPLOAD_STATUSES = [
  'uploading', 'finalizing', 'cancelling', 'uploaded',
  'upload_failed', 'aborted', 'expired',
] as const

export type PublicUploadStatus = (typeof PUBLIC_UPLOAD_STATUSES)[number]
export type MediaKind = 'image' | 'video'

export function planUploadParts(sizeBytes: number) {
  if (!Number.isSafeInteger(sizeBytes) || sizeBytes < MIN_FILE_SIZE_BYTES || sizeBytes > MAX_FILE_SIZE_BYTES) {
    throw new RangeError('file size is outside the supported range')
  }
  const count = Math.ceil(sizeBytes / PART_SIZE_BYTES)
  return Array.from({ length: count }, (_, index) => ({
    partNumber: index + 1,
    offsetBytes: index * PART_SIZE_BYTES,
    sizeBytes: Math.min(PART_SIZE_BYTES, sizeBytes - index * PART_SIZE_BYTES),
  }))
}
```

Create TypeBox schemas with `additionalProperties: false` for the request/response JSON in API sections 2, 5, and 6, satisfying the already-failing schema cases without weakening them.

```ts
import { createHash } from 'node:crypto'
import { readdir, readFile } from 'node:fs/promises'
import { join } from 'node:path'

export async function runMigrations(pool: Pool, directory: string): Promise<void> {
  const client = await pool.connect()
  try {
    await client.query('select pg_advisory_lock($1)', [1_977_071_401])
    await client.query(`create table if not exists public.schema_migrations (
      version text primary key, checksum text not null, applied_at timestamptz not null default clock_timestamp()
    )`)
    const files = (await readdir(directory))
      .filter((name) => /^\d{4}_[a-z0-9_]+\.sql$/.test(name))
      .sort()
    for (const version of files) {
      const sql = await readFile(join(directory, version), 'utf8')
      const checksum = createHash('sha256').update(sql).digest('hex')
      const existing = await client.query<{ checksum: string }>(
        'select checksum from public.schema_migrations where version = $1',
        [version],
      )
      if (existing.rowCount === 1) {
        if (existing.rows[0].checksum !== checksum) {
          throw new Error(`applied migration changed: ${version}`)
        }
        continue
      }
      await client.query('begin')
      try {
        await client.query(sql)
        await client.query(
          'insert into public.schema_migrations(version, checksum) values ($1, $2)',
          [version, checksum],
        )
        await client.query('commit')
      } catch (error) {
        await client.query('rollback')
        throw error
      }
    }
  } finally {
    await client.query('select pg_advisory_unlock($1)', [1_977_071_401]).catch(() => undefined)
    client.release()
  }
}
```
